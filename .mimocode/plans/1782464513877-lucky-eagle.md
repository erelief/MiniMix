# 计划：图章颜色默认值 + 清除画布重置序号

## 需求
1. 清除画布所有图片时，序列号工具的 `nextNumber` 重置为 1
2. 图章工具：√ 默认绿色，× 默认红色；切换形状时自动切换对应颜色；重开软件恢复默认

## 修改文件

### 1. `annotation.js` (默认设置)
**位置**: 行 119-125，`stamp` 默认配置

添加 `checkColor` 和 `xColor` 字段，保留 `color` 作为当前活跃颜色：
```js
stamp: {
  shape: 'check',
  size: 256,
  color: '#00A651',        // 当前颜色（√ 默认绿）
  checkColor: '#00A651',   // √ 的默认绿色
  xColor: '#E61919',       // × 的默认红色
  opacity: 100,
  shadow: 35,
},
```

### 2. `floating-toolbar.js` (形状切换时自动切换颜色)
**位置**: 行 1410-1421，`buildStampMenu` 中 check/x 按钮的 click handler

在切换 shape 时，同时将 `color` 设置为对应形状的默认颜色：
- 选 √：`onChange('stamp', 'color', s.checkColor)` + `onChange('stamp', 'shape', 'check')`
- 选 ×：`onChange('stamp', 'color', s.xColor)` + `onChange('stamp', 'shape', 'x')`

### 3. `main.js` (清除画布重置序号)
**位置**: 行 985-997，`btnClear` click handler

在 `pushUndo()` 之后、`recomputeAndRender()` 之前添加：
```js
state.toolSettings.sequence.nextNumber = 1;
```

## 验证
1. 打开 app，进入编辑模式，选图章工具 → 默认应为 √ 形状、绿色
2. 切换到 × → 颜色应自动变为红色
3. 切换回 √ → 颜色应自动变回绿色
4. 手动改 √ 颜色为蓝色，切换到 × 再切回 √ → 应保持蓝色（会话内记忆）
5. 重开 app → √ 恢复绿色、× 恢复红色
6. 添加图片，放置序列号标注（如 1, 2, 3），清除画布，重新添加图片，放置序列号 → 应从 1 开始
